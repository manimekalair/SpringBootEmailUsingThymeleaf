Project Management and Source Control

XMLSpy® Professional and Enterprise Editions provide project management and source control support to help you organize and manage your work. The XMLSpy® Project Management & Source Control capabilities are easy-to-use applications that keep your creations in order, so you can stay ahead.

Using the project view you can group your files and frequently used URLs into logical folders in an arbitrary structure. The folders may correspond to actual directories on your file system, or you can define file-type extensions for each folder so that common files are kept together in one convenient place. These project folders are virtual folders and do not need to correspond to any hierarchical organization of the files on your hard disk.

You can also add external folders from your hard disk or WebDAV servers without actually adding the individual files. As such, the files and sub-folders will be shown as they are on the disk or WebDAV server. This capability is useful when you need to perform the same operation on a large number of files contained in one or more real folders, because it saves you from having to explicitly add them all to the XMLSpy® project system.






By assigning different XSL transformation parameters to each folder you can have the same physical file appear in more than one project folder. This is especially useful when you want to keep your data in one XML file and use different XSL stylesheets to produce different output (e.g., separate HTML and WML presentations).

You may want to assign a DTD and an XML Schema to different project folders. This will permit you to validate a file against both the DTD and the schema without changing the file itself, and can be very helpful when you are in the process of transitioning your content modeling from DTDs to XML Schemas.

The ‘Project Properties’ page provides you with the flexibility to specify special processing options for each designated folder.






Whenever you create, receive, or download a new XML document, you can automatically add the active file and all related files to a project folder. The integrated parser will automatically locate any schema, DTD, or external entity reference and add all the related files or URLs to the project. If you have designated different file-types for different folders, the project management capability will automatically put the newly found files in their proper folder locations. Of course, you can also simply add a single file, or group of files, from the available ‘Browse’ dialog.

Once you have built your project, you can apply most commands to the entire project, or a selected group of files or folders, in a batch-like operation. Such commands include:

Check well-formedness
Validate
XSL transformation
Generate Schema/DTD
Send email

The source control facility lets you tie your project to a centralized version control or software configuration management (SCM) system, such as Microsoft Visual SourceSafe and compatible systems like CVS, PVCS, and StarTeam. Once your project is affiliated with a version control system, XMLSpy® automatically monitors the status of all files and prompts you to check out a file whenever you start to modify it. The state of each file is indicated with checkmarks or locks in the upper right corner of each file icon.






This functionality allows a team to work collectively on a set of XML documents, ensuring proper access and version control. Since most version control systems support many document types, seamless management of XML documents, schemas, and XSLT stylesheets can be achieved using the same proven tools already commonly used by C++, Visual Basic, Java, and Web developers.
