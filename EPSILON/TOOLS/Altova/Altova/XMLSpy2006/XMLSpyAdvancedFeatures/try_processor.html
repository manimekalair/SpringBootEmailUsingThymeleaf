External XSL/XSL:FO Processor Support

XMLSpy® Professional and Enterprise Editions feature an extensible XML Transformation Engine interface that allows you to either use Altova’s built-in XSLT processor, which now supports XSLT 1.0 and XSLT 2.0, or plug in an external XSLT, XSL:FO, or related stylesheet processor of your choice to perform XML transformations.

When working in enterprise environments, it is often the case that groups of developers will standardize on a particular XSLT processor for all server systems throughout the organization. XMLSpy® fully supports this trend, letting you use that same processor in your development environment. Via the XSL Options dialog you can pick either the built-in Altova processor, MSXML, or any other external processor that can be driven from a command-line interface.






To render XML data into a visual representation, the Extensible Stylesheet Language (XSL) defines a set of Formatting Objects (XSL:FO) that allow XML documents to be transformed into PostScript, PDF, or any other page layout language. With XSL:FO processor support you immediately gain access to direct XSL:FO transformations from within XMLSpy® using either Apache FOP or any other third-party XSL:FO processor. In just a click of a button, you can transform your XML files into PDF documents using XSL:FO stylesheets.

The ability to use external stylesheet processors is crucial in these and many other situations. Just think what the XMLSpy® XSL/XSL:FO External Processor Support could do for you, and give it a try today!
