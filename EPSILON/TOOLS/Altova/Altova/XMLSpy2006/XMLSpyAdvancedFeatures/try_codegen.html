Code Generation Capability

XMLSpy® Enterprise Edition provides an extremely powerful code generation capability that virtually eliminates the programming effort associated with XML application development projects. The XMLSpy® Code Generation Capability automatically generates Java, C++, or Microsoft C# class files based on data elements defined in your XML Schema, dramatically improving accuracy and saving precious time.

In most XML application scenarios an XML document must be bound to an external software application or runtime environment such as a business-to-business application or Web service. Binding XML data elements to related software objects this way allows you to work programmatically with the code from within your software application.

Implementing an XML data binding usually involves writing hundreds or even thousands of lines of infrastructure code to load, edit and save XML documents via low-level XML processing APIs. In addition to this tedious, error-prone task, a data binding also requires additional higher level functions to validate, process, transform, and perform other common in-memory operations on XML documents. XMLSpy® automates these chores through a simple menu item that generates all the necessary program code based on your XML Schema’s inherent support for datatypes and object oriented design. Automating the data binding process accelerates your development project, resulting in substantial cost savings and time to market advantages.

Once you have modeled your data elements using the XMLSpy® schema editor, you can auto-generate data bindings, or class files, corresponding to elements defined in the data model. Simply pick your programming language and select your settings, and the code will be instantly generated and stored in the directory you desire.






Default templates are provided to ensure that your generated class definitions correspond accurately to all declared elements or complex types in your XML Schema. The templates support industry standard XML processing APIs including Microsoft MSXML, Java API for XML Processing (JAXP), and Microsoft System.XML. Generated C++ output includes a Visual Studio 6.0 project file, while generated Java output includes a Sun Forte for Java project file and Apache Ant build script, and Microsoft C# output includes a Visual Studio.NET project file. Generation of project files for Borland C# Builder, MONO Makefile and other third party IDEs is also supported.






Furthermore, a simple yet powerful template language gives you the ability to customize the existing templates or build your own templates to automate the generation of anything imaginable, e.g., SQL scripts, ASP or WML code. This allows you to customize your output code by binding XML Schema datatypes to the primitive datatypes of your particular programming language.

Just imagine how much you will save when you need to modify the rules defined in your XML Schema. Changes to your schema can be immediately reconciled with a software implementation by simply re-running the code generator. When numerous XML documents need to be updated according to the modified schema, the code generator can produce the basis of your programmatic update process in a snap.

The XMLSpy® Code Generator Capability automates the writing of essential infrastructure code, freeing you to focus on implementing critical business logic. Give it a try today!
