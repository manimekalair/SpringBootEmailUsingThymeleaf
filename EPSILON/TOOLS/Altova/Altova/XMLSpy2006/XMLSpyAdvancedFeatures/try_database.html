Database/Text Import/Export

XMLSpy® Enterprise and Professional Editions feature powerful import and export capabilities for all major databases and legacy text files. While XML is ideally suited for today’s information and data exchange scenarios, much of the data we still use exists only in database or text form. The XMLSpy® Database/Test Import/Export functionality overcomes interoperability challenges, allowing you to easily convert back and forth between database or text sources and XML files.

The database import/export functionality interacts natively with the most popular relational databases, including Microsoft Access, SQL Server, Oracle, MySQL, Sybase, and IBM DB2. Other databases are also supported using ADO or ODBC connectivity. With these new capabilities, you can import data from a relational database into an XML document, generate an XML Schema based on a relational database, export data from an XML document to a database, and generate a database structure from an XML Schema.

Like all the other database capabilities in Altova’s product line, the database import function allows you to select your specific database type or specify any other ADO or ODBC-conformant database.









There are two options for importing data from a database into an XML document. The first option is great for simple XML projects that don’t require a schema or are based on a DTD. With this option you select a table and pull in raw data in its basic tabular format. The second option works in conjunction with the ‘Create XML Schema from DB Structure’ function and allows you to import database data maintaining all the relationships and dependencies defined in the content model.

Although they produce different results, both database import methods operate in a similar fashion. You can choose representations for date and number formats, select import defaults, and decide how to handle primary/foreign keys before executing the actual import process. Data is converted in the blink of an eye once the ‘Import’ button is clicked.






Importing text files into XML documents is very straightforward. Just choose your file to convert and pick the conversion settings and you’re good to go.






Creating an XML Schema from a database is just as easy and versatile as importing database data. After specifying the source database type and filename, you can select the tables, views and system tables upon which to base the schema, and pick your schema options. That’s all there is to it. Click the ‘Create Schema’ button, and your XML Schema appears before your eyes, perfectly formatted and ready to use. The XML Schema will properly reflect the 1:n relationships expressed in your relational database and contain the corresponding identity constraints.






When you need to export XML data into a database, it’s as simple as specifying where to start, how to handle export fields, and which elements to include. Then just select which database type you wish to append or create with your data, and you’re done. The data is instantly converted and stored in your database. Exporting XML data into text files works practically the same way.






For times when you want to define a more intricate database with the same rules as an established XML Schema, XMLSpy® Professional and Enterprise Editions offer the best solution. Simply select your database type from the comprehensive list then open a database to modify, or create a new one from within XMLSpy®. If your schema includes identity constraints, they will automatically transfer to your new database structure. Otherwise, you can easily point and click to define relationships between elements. You can preview the structure and even edit and save the auto-generated SQL statement associated with your conversion. As with all of the database import/export capabilities, the conversion process is as fast as lightning. When you click ‘Create Tables’ your new database structure is ready in a flash.
