SOAP Debugger and WSDL Editor

XMLSpy® Enterprise Edition provides advanced Web services support, including a powerful SOAP (Simple Object Access Protocol) debugger, a SOAP and WSDL (Web services Description Language) client interface, and a full blown WSDL editor. These capabilities allow you to interpret WSDL documents, create SOAP requests, submit them to a Web service, and view the SOAP response, ensuring proper functionality.



The XMLSpy® SOAP Debugger acts as a Web services proxy between a Web services client and server. This allows you to inspect WSDL files and step through Web services transactions examining every request and response XML document. It also lets you set breakpoints on SOAP functions (on the request or response), and you can even define conditional breakpoints that trigger when a certain request or response contains data that has been specified in an XPath query.






This is a very practical tool that lets you tackle real-world Web services development on all of the major Web services platforms, including Microsoft .NET, Eclipse, and Java 2 Enterprise Edition.

Using the XMLSpy® SOAP and WSDL Client Interface you can open any existing WSDL document and immediately learn about the functions provided by the corresponding Web service. Selecting a function automatically creates a SOAP request message, fills in the data, and sends it to the Web service. Then, the SOAP response message is received and displayed. This feature includes complete support for highly structured data (e.g., using complexTypes as parameters or in the result set).






The XMLSpy® WSDL Editor allows you to edit, visualize, and validate any WSDL file. WSDL is ideally suited as an interface definition language (IDL) for architecting Web services applications. By first building an interface, client and server programmers can implement their respective projects using any language or operating system, avoiding interoperability problems.






The WSDL editor even offers a documentation generation utility that makes it easy for Web services developers to document and publish a Web services interface for business partners, other developers, or the public. Any WSDL file (including auto-generated files used by certain server platforms) can be easily annotated then published in a Microsoft Word or HTML output file.
