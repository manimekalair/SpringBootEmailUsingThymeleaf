XML Differencing

XMLSpy速 Professional and Enterprise Editions include a visual XML differencing engine for graphically comparing and combining XML files and directories. The XMLSpy速 XML Differencing Engine allows you to easily compare XML documents and directories, identify differences, select parts for inclusion in the new version, and then perform an XML-aware file merge.






This XML differencing functionality is completely customizable. You can specify if entities should be resolved, if namespace prefixes, white space, CDATA, processing instructions, comments, or the ordering of attributes/child elements should be considered or not, and how to visualize differences.






Projects developed in a team environment are much easier to manage with this differencing tool because it allows you to quickly identify which parts of the code have changed, saving time and effort in editing, troubleshooting, and versioning.

The differencing engine lets you compare entire directories and generate a report detailing the variation. This allows you to quickly ascertain which files are new, deleted, or modified. Clicking on an entry opens the two related files and compares them visually.






Conventional differencing utilities report line numbers corresponding to parts of the code that have been inserted, deleted or modified. Though this may be sufficient when working with conventional programming languages, XML technologies are unique and require more robust differencing utilities. The XML-awareness of this differencing engine makes it the best tool in many important scenarios:

Suppose an XML developer changes the default namespace prefix in an XML instance document. This change does not materially affect the contents of the XML document, and while a conventional software differencing utility would report that virtually every line of code in the XML instance document has changed, the XML differencing engine available in xmlspy can deal with this change appropriately.
The order of XML attributes is irrelevant because XML processors do not consider the sequence that attributes appear in a particular element. Our XML differencing engine can account for this and intelligently ignore the attribute order, but a conventional differencing utility cannot and would therefore report every change in attribute ordering.
There are instances when the ordering of child elements is irrelevant, for example, if a schema defines a relationship using an "all" compositor, the ordering of child elements is immaterial. Our XML differencing engine can be set to overlook irrelevant differences, but a conventional differencing utility cannot.
An XML processor resolves entity references at run-time, so although an XML file may not have changed, the entity definitions could have, resulting in a different XML file. A conventional software differencing utility would not pick up on this, but our XML differencing engine would.
